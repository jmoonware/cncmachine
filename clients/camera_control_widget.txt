Create a USB Camera graphical control widget in Python suitable to run on a Raspberry Pi 4 for the USB camera IDS 3680XLE-M, a monochrome camera. The GUI control should use PyQt6. The camera control code should us the ids_peak python libraries, examples of using this library can be found in https://github.com/ids-imaging/ids-peak-examples/tree/main/python. The control should have a displayed image area at the top of the window. The following elements should also be implemented in this widget, below the display image:
* A Start/Stop Checkbox to start/stop continuous image streaming to the displayed image area
* A Capture Button to capture a single image. The Capture Button opens a new window that displays the single image, and has a button to save the image. If the "save" button is pressed, a file dialog opens to choose a name and place to save the image (defaulting to 'image.png' and the current directory)
* A numeric input box for setting exposure time in microseconds (default 5000, increment of 1). Changing this value changes the USB camera's exposure time.
* A numeric input box for gain (default 1, increments of 0.1). Changing this value changes the USB camera's gain. Minimum and maximum values can be obtained via querying the NodeMap using FindNode("Gain").
* Mouse driven cross-hairs that "slice" a single vertical and horizontal line in the displayed image area. The position of each line should be displayed below the image as the cross-hairs move. Right-clicking the mouse causes the current pixel X-Y position of the cross-hair to be set to 0,0, and the displayed position values should be updated using that origin. A line graph of the pixel intensity values for each cursor line should be displayed to the right of the displayed image. Units of each graph and cursor position should be in pixels by default. A check-box indicates whether to use a scale factor to convert the graph abcissas and displayed cross-hair values to microns, where the scale factor from pixels to microns is an adjustable value (default is 5 microns/pixel.) 
* Add a Python-callable function that acquires a single image (with exposure and gain parameters as arguments). The function should return the image data as a 2-d Numpy integer array.
