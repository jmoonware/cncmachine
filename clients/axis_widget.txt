Create a graphical 'axis widget' in python using PyQt6 for controlling an axis of motion. The mechanical axis and motors are controlled by a Parker 6k8 controller. The command reference for this controller is found here: https://www.parkermotion.com/manuals/6k/6K_Command_Reference.pdf. 

The Python widget will run on a Raspberry Pi 4 and communicate with the Parker 6k8 via a 9600 baud serial port on /dev/ttyUSB0. The serial port is connected via a three-wire connection that does not use hardware low control. It also does not use software  (XON/XOFF) flow control. The serial port can be shared across multiple axis widget controls, and the widget should be designed to be incorporated into a larger program with multiple instances of this axis widget. As multiple threads may be sending/receiving commands to/from the Parker 6k8, be careful that sending one command and receiving the Parker response one-at-a-time can be executed in a thread-safe manner.

The axis number should be configurable (default = 5) in code. Updated values of numeric controls should be sent to the Parker 6k8 when the control loses focus or enter is hit (not just when the value in the box changes.)
 
Each axis widget should have the following controls for a particular axis:
* A label identifying the axis (both a numeric value and a configurable human readable string, such as 'Z')
* An integer input box (in steps) for the distance to travel. This value is set using the D command (e.g., if the position box value is 1, and the axis is 5, then 5D1 is sent). Default is 5000.
* A label displaying the physical distance calculated from the value in the distance input box. The physical distance is calculated from a microns/step parameter, configurable in code. The microns per step parameter is calculated from the number of steps per revolution (default 8000) and the stage thread pitch (default 10000 microns.)
* An integer input box (in rev/s) for velocity of movement. Default is 1.
* An integer input box (in rev/s/s) for acceleration of movement. For example, if this box contains the value 1, and X=5, this transmits 5A1 to the Parker 6k8. Default is 1.
* An integer input box (in rev/s/s) for deceleration of movement. Default is 1.
* An Absolute check box (default unchecked) If checked, the distance is interpreted as an absolute encoder position (e.g. for axis 5, via the 5M1 command) Otherwise the the 5M0 (incremental mode) command is sent when the box is unchecked.
* A right arrow button that executes a positive move based on the above motion settings, and a left arrow button which executes a negative move based on the above motion settings (basically, a GO command for the axis, e.g., if axis 5, a 5GO command). An additional D command may need to be sent before the GO command to adjust the sign of the value held in the distance input box. For instance, if the value in the Distance box is 5000, the 5D-5000 command should be sent if the negative arrow is hit, before sending the 5GO command. The right arrow on the keyboard should press the right arrow button on the gui; similarly for the left arrow. The keyboard keys that are used for activating the right or left arrow  
* A 'Home' button that performs a homing command for this axis. For example, if this is axis 5 then the 5HOM<DIR> command is sent, where <DIR> is either 1 or 0, determined by the 'Home Direction' checkbox state (see next item.) 
* A checkbox labeled 'Home Direction'. When checked, causes the homing direction to be positive (e.g., 5HOM1). If unchecked, the command 5HOM0 is sent.
* A set of indicators (gray or green) that use the output of the Transfer Axis Status (TAS) command. The Parker6k8 should be polled with the TAS command about once per second. The TAS command reports 32 values for each active axis (as a string of 0's or 1', where 0 should be linked to gray, and 1 to green on the indicator.) The following bits should be displayed for the appropriate axis in the indicators as follows (bit#, label): 1,Moving; 2,NegPos; 5,Homed; 12,Stall; 13,Shutdown; 14,Fault; 15, PosLim; 16, NegLim; 23, PositErr;
* A 'Halt' button that sends an axis Kill command. For example, to kill axis 5 the command to send is !K00001000.
* A label displaying the current encoder Position (as returned from the TFB command) This should be included as part of the status TAS polling loop.
* A Zero button which uses the PSET command to set the Parker6k8 reported encoder value to zero. For example, for axis 5, the command to send is 5PSET0.
* A 'Drive On' checkbox that, when checked, turns on the axis drive. For instance, for Axis 5 the command sent when checked is 5DRIVE1, when unchecked 5DRIVE0 
* A Response label that shows the last response from the Parker 6k8. Usually this is '>' plus a line-feed/carriage return. If an error occurs the response starts with '?'. 
* All commands sent to and received from the Parker6k8 should be logged to file using the Python logging library at DEBUG level of logging. At INFO level, the polling TAS and TFB commands are not logged. Responses from the Parker 6k8 that start with '?' are logged as errors. The there should be 10 rotating log files each  limited to 1 MB.
